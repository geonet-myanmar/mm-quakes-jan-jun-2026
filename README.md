# Myanmar Seismic Analytics Dashboard (Jan – Jun 2026)

An interactive, high-fidelity spatial and non-spatial analytics dashboard designed to analyze and visualize earthquake patterns in Myanmar. The project maps 57 seismic events recorded between January 14, 2026, and June 15, 2026, and correlates them with lineaments from the 2011 Tectonic Map of Myanmar.

**Live Demo:** [https://geonet-myanmar.github.io/mm-quakes-jan-jun-2026/](https://geonet-myanmar.github.io/mm-quakes-jan-jun-2026/)

---

## 🚀 Key Features

* **Interactive GIS Map**: Powered by Leaflet.js, featuring dark-themed vector and satellite imagery base maps.
* **Proportional Epicenters**: Earthquake markers are sized exponentially by magnitude and color-coded by hypocentral depth (Cyan = Shallow, Orange = Intermediate, Magenta = Deep).
* **Tectonic Layer**: Visualizes `Myanmar_Tectonic_Map_2011.geojson` as a single-class lineament background with hover highlights and detailed attribute popups.
* **Dynamic Cross-Filtering**: Sliders for Magnitude, Depth, and Fault Proximity, combined with date bounds and location search, filter the map, KPI cards, charts, and table dynamically.
* **Geological Convenioned Scatter Plot**: Plots Magnitude against Depth with an inverted Y-axis, showing deeper hypocenters at the bottom.
* **Seismic Ledger**: A sortable, scrollable table of all matching events, with center-and-zoom controls that focus the map on specific epicenters.
* **Glassmorphic Design System**: Styled with modern, high-contrast, semi-transparent CSS components and smooth slide-in entry animations.

## 📂 Project Architecture

* **[`index.html`](index.html)**: Main HTML5 structure for filters, KPI metrics, map canvas, tabs, and ledger table.
* **[`index.css`](index.css)**: Vanilla CSS design system defining glassmorphism, responsive grids, custom input sliders, and web layouts.
* **[`app.js`](app.js)**: Javascript controller handling Leaflet mapping, Chart.js renderings, event listeners, and filtering logic.
* **[`process_spatial_data.py`](process_spatial_data.py)**: Enriches the raw earthquake data with spatial relationships (nearest fault lineament, nearest named fault, magnitude classes, and depth classes).
* **[`analyzed_earthquakes.json`](analyzed_earthquakes.json)**: The enriched GeoJSON-like database generated by the spatial analysis script.
* **[`dmh_myanmar_earthquake_data.geojson`](dmh_myanmar_earthquake_data.geojson)**: Raw input earthquake coordinates and properties from DMH.
* **[`Myanmar_Tectonic_Map_2011.geojson`](Myanmar_Tectonic_Map_2011.geojson)**: Tectonic lineament polyline background layers.
* **[`server.py`](server.py)**: Quiet local HTTP server wrapper for testing the application locally.

## 🧮 Spatial Analysis Methodology

To evaluate seismic risk without requiring heavy database dependencies, the data was pre-processed using a local flat projection in **`process_spatial_data.py`**:

1. **Equirectangular Projection**: Epicentral and fault vertices are projected onto a local flat plane centered on Myanmar in kilometers using:
   $$y = \text{Latitude} \times 110.574 \text{ km/deg}$$
   $$x = \text{Longitude} \times 111.320 \times \cos\left(\text{Latitude} \times \frac{\pi}{180}\right) \text{ km/deg}$$
2. **Point-to-Segment Calculation**: For each epicenter $P$, the shortest distance to every line segment $AB$ of the tectonic map is calculated using vector projection:
   $$t = \max\left(0, \min\left(1, \frac{\vec{AP} \cdot \vec{AB}}{\|\vec{AB}\|^2}\right)\right)$$
   The nearest point $C$ on the segment is $A + t\vec{AB}$, and the Euclidean distance $\|\vec{PC}\|$ yields the distance to the fault in kilometers.
3. **Data Enrichment**: This distance along with name, code, segment name, magnitude, and depth classifications are appended to the earthquake properties.

## 💻 Running Locally

### Prerequisites
* Python 3.x

### Steps
1. Clone the repository:
   ```bash
   git clone https://github.com/geonet-myanmar/mm-quakes-jan-jun-2026.git
   cd mm-quakes-jan-jun-2026
   ```
2. Start the local server:
   ```bash
   python server.py
   ```
   This will spin up a local HTTP server at [http://localhost:8000](http://localhost:8000) and automatically open the dashboard in your default browser.

## 🌐 Deploying to GitHub Pages

The project is fully configured for static deployment on **GitHub Pages**:
1. Go to the repository settings on GitHub.
2. Navigate to **Pages** in the sidebar.
3. Under **Build and deployment**, select **Deploy from a branch**.
4. Set the branch to `main` (or the branch you pushed to) and the folder to `/ (root)`.
5. Click **Save**. The dashboard will be published at `https://geonet-myanmar.github.io/mm-quakes-jan-jun-2026/`.
